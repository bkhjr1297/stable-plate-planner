# Annabelle Zero-Dollar Venture Framework

Use this as the repeatable weekly launch system for every $0 upfront venture.

## Weekly cadence

1. **Monday — choose the wedge:** one audience, one pain, one free artifact, one manual paid offer.
2. **Tuesday — build the free artifact:** static page/tool/checklist/template only; no paid APIs, inventory, or ads.
3. **Wednesday — add SEO + content spine:** landing page metadata, sitemap entry, blog category, 5 starter keywords.
4. **Thursday — publish and verify:** GitHub Pages or another free host; test all CTAs and disclaimers.
5. **Friday — distribution:** share ethically in 3 relevant free communities; ask for feedback, not hype.
6. **Weekend — fulfillment and decision:** deliver any manual orders, collect objections, decide keep/kill/iterate.

## Required files per venture

- `index.html` — clear promise, free tool/resource, CTA, safety boundary if needed.
- `blog/` — daily practical SEO posts.
- `sitemap.xml` and `robots.txt` — search discovery basics.
- `framework/` or `README.md` — offer, validation checklist, fulfillment workflow.
- `.github/workflows/` — free automation when useful.

## Offer rules

- Start with one manual service between $9 and $49.
- Keep delivery simple: email, DM, PDF, spreadsheet, or checklist.
- No sensitive customer data in public GitHub Issues.
- For health, finance, legal, or housing topics: educational support only, conservative claims, strong disclaimers.

## Validation scorecard

Continue only if at least one signal appears within 7 days:

- 5+ real users try the free tool/resource.
- 2+ people ask for a custom version.
- 1+ paid order or clear buying intent.
- Repeated search/community questions match the exact pain.

If no signal, preserve the framework and launch the next venture rather than overbuilding.
